Agenda

为什么网页更适合作为“高级 PPT”

把 PPT 的“线性翻页”保留，同时拿到 Web 的“非线性能力”。

01 · 结构
  • slide 分段 + scroll-snap
  • hash 深链接：可分享某一页
  • 总览模式：像 Keynote 的缩略图
02 · 表达
  • 动效按“进入视口”触发
  • 交互组件：Tab / Step / Hover
  • 可嵌入真实数据与可交互图表
03 · 工程化
  • 主题变量：一处改色全局生效
  • 组件可复用，团队协作友好
  • 可导出打印（PDF）与版本管理

滚轮节流翻页（示意）
// wheel -> next/prev slide (with cooldown)
let lock = false;
window.addEventListener('wheel', (e) => {
  if (lock) return;
  if (Math.abs(e.deltaY) < 18) return;
  e.preventDefault();
  lock = true;
  goTo(current + (e.deltaY > 0 ? 1 : -1));
  setTimeout(() => (lock = false), 680);
}, { passive: false });
视口触发动效（示意）
const io = new IntersectionObserver((entries) => {
  entries.forEach((en) => {
    en.target.classList.toggle('in-view', en.isIntersecting);
  });
}, { threshold: 0.35 });

document.querySelectorAll('.slide').forEach(s => io.observe(s));

FAQ

常见问题

这类交互在 PPT 很难“自然地做”。

是否必须用框架（React/Vue）？
不必须。你现在这套是纯原生三件套，方便交付与迁移。
如何做到更“演讲模式”？
增加计时器、激光笔、隐藏 UI、双屏 presenter view 等。
如何给每页加更多动效？
建议按“进入视口”触发，并提供 per-slide 动效类名开关。
